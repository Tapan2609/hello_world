Test API documentation

/v1/start

This will trigger the test execution within the container

/v1/start get

get

get /v1/start

  • Response

HTTP status code 200

Body

Type: application/json

Example:

{
  "message": "started"
}

/v1/summary

At any point, accessing this api will return the summary of testcases executed till that point

/v1/summary get

get

get /v1/summary

  • Response

HTTP status code 200

Body

Type: application/json

Example:

{
  "total": 10,
  "passed": 7,
  "failed": 1,
  "skipped": 2
}

/v1/end

This will end the testcase execution within the container and it will return to an idle state

/v1/end get

get

get /v1/end

  • Response

HTTP status code 200

Body

Type: application/json

Example:

{
  "message": "ended"
}

/v1/live

This api will return all the test cases that were executed from the last time this api was accessed. Please note that this is not cumulative and returns on the delta of testcases that were executed after the last call to this api.

/v1/live get

get

get /v1/live

  • Response

HTTP status code 200

Body

Type: application/json

Example:

[
  {
    "name": "testcase1",
    "suitename": "suite1",
    "status": "passed"
  },
  {
    "name": "testcase2",
    "suitename": "suite1",
    "status": "failed"
  },
  {
    "name": "testcase3",
    "suitename": "suite2",
    "status": "skipped"
  }
]

/v1/status

This api will report the status of the test container any given point in time. It can one of the following values "running", "stopped"

/v1/status get

get

get /v1/status

  • Response

HTTP status code 200

Body

Type: application/json

Example:

{
  "status": "running"
}
